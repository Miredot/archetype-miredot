General

The ${serviceName} has the following connectors for Requests and Responses that are implemented with JSON bodies.

  • X Connector : This is used for X.
  • Y Connector : This is used for Y.
  • ...

Media types

The Media Type for new APIs are: 

    Accept: application/vnd.acme.#your package#-#major#.#minor#+json
    Content-Type: application/vnd.acme.#your package#-#major#.#minor#+json

The Media Type for deprecated APIs are: 

    Accept: application/vnd.acme.#your package#-#major#.#minor#+json
    Content-Type: application/vnd.acme.#your package#-#major#.#minor#+json

    Accept: application/vnd.acme.#your package#-#major#.#minor#+json
    Content-Type: application/vnd.acme.#your package#-#major#.#minor#+json

The default Media type is: 

    Accept: application/vnd.acme.#your package#-#major#.#minor#+json
    Content-Type: application/vnd.acme.#your package#-#major#.#minor#+json

Service availability (ping)

The service can be validated using the HEAD method for all resources that implement the GET method. The HEAD method also performs authentication. A successful request returns HTTP status code 200 and no response body.
